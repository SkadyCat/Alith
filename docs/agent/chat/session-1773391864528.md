支持继承/扩展的主流数据协议/格式：

| 格式 | 继承方式 | 说明 |
|------|----------|------|
| **JSON Schema** | `$ref` + `allOf` | JSON 的 Schema 层支持组合继承，JSON 数据本身不支持 |
| **YAML** | Anchors (`&`) + Merge (`<<:`) | 原生支持引用和合并，常用于 CI/配置文件 |
| **Protocol Buffers (proto3)** | 无真正继承，但有 `oneof`/嵌套 message | 结构化强，不含继承语义 |
| **XML / XSD** | `xs:extension`、`xs:restriction` | 完整继承支持，SOAP/企业系统常用 |
| **TypeScript interface/type** | `extends`、交叉类型 `&` | 编译时类型继承，生成的 JSON 无感知 |
| **OpenAPI 3.x** | `allOf` / `anyOf` / `discriminator` | 支持多态继承，基于 JSON Schema |
| **Cap'n Proto** | `union` + interface 继承 | RPC 层支持服务继承 |
| **Thrift** | `extends`（service 层） | service 可继承，struct 不支持 |
| **Avro** | `record` 嵌套 + `union` | 无经典继承，通过联合类型模拟 |

**最推荐的选择：**

- **配置文件场景** → **YAML**（最直接，锚点 + 合并键）
- **接口/类型定义** → **JSON Schema + allOf** 或 **TypeScript**
- **数据交换协议** → **XML/XSD**（完整继承，但重）或 **OpenAPI**
- **高性能二进制** → **Protocol Buffers**（无继承但有组合）

**YAML 示例：**
```yaml
base: &base
  timeout: 30
  retries: 3

prod:
  <<: *base      # 继承 base
  timeout: 60    # 覆盖
```
