# SOP：UI 编辑器设计规范维护

**触发词**：ui规范 | 设计规范 | uieditor | 控件选择 | 进度条 | 重叠 | 布局

---

## 执行步骤

1. **读取规范**：读取 `docs/uieditor-thinking.md` 了解当前已记录的设计问题
2. **分析问题**：理解用户指出的具体设计误区（控件误用、语义不清、结构冗余等）
3. **定位 session**：在 canvas-editor 中找到对应 session 文件，查看实际控件结构
4. **追加条目**：在 `docs/uieditor-thinking.md` 的"问题记录"区追加新问题条目，包含：
   - 错误设计（用树状图表示控件结构）
   - 问题分析（为什么不好）
   - 正确设计（应该使用的控件）
   - 规则（一句话总结）
5. **更新检查清单**：将新问题对应的检查项加入"设计检查清单"
6. **（可选）修复 session**：如果能确定正确做法，在 session 文件中修复控件结构
7. **验证**：确认文档保存成功，进入 POLL

## 注意事项

- `uieditor-thinking.md` 是面向人类开发者的设计规范，语言应清晰易懂
- 问题编号递增，不要重复
- 如果问题涉及 canvas-editor 代码（如缺少控件类型），需同步修改 `elements.json` 或 `app.js`
