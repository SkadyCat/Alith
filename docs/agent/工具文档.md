## 一、执行工具 `/tools`

### `POST /tools/python` — 执行 Python 代码

在服务端沙箱中执行任意 Python 代码片段，支持 `requests`、`re`、`math`、`datetime` 等常用模块，结果以 JSON 返回。

**请求体**

| 字段      | 类型     | 必填 | 说明                                    |
|-----------|----------|------|-----------------------------------------|
| `code`    | `string` | ✅   | Python 代码字符串                       |
| `timeout` | `number` | ❌   | 超时毫秒，默认 `30000`，最大 `120000`  |

**响应**

| 字段       | 类型      | 说明                             |
|------------|-----------|----------------------------------|
| `success`  | `boolean` | `exitCode === 0` 时为 `true`     |
| `stdout`   | `string`  | 标准输出                         |
| `stderr`   | `string`  | 标准错误 / 异常堆栈              |
| `exitCode` | `number`  | 退出码（0 表示成功，-1 超时）    |

**示例**

```bash
curl -X POST http://localhost:7439/tools/python \
  -H "Content-Type: application/json" \
  -d '{
    "code": "import datetime\nprint(datetime.date.today())"
  }'
```

```json
{ "success": true, "stdout": "2026-03-02\n", "stderr": "", "exitCode": 0 }
```

**内置模块**：`re` `math` `os` `sys` `time` `datetime` `hashlib` `base64` `urllib` `json` `collections` `itertools` `requests`（可选）

---

### `POST /tools/shell` — 执行 PowerShell 命令

在服务端执行 PowerShell 5.1 命令或多行脚本，结果以 JSON 返回。

**请求体**

| 字段      | 类型     | 必填 | 说明                                    |
|-----------|----------|------|-----------------------------------------|
| `command` | `string` | ✅   | PowerShell 命令或脚本字符串             |
| `timeout` | `number` | ❌   | 超时毫秒，默认 `30000`，最大 `120000`  |

**响应**

| 字段       | 类型      | 说明                          |
|------------|-----------|-------------------------------|
| `success`  | `boolean` | `exitCode === 0` 时为 `true`  |
| `stdout`   | `string`  | 标准输出                      |
| `stderr`   | `string`  | 错误信息                      |
| `exitCode` | `number`  | 退出码                        |

**示例**

```bash
curl -X POST http://localhost:7439/tools/shell \
  -H "Content-Type: application/json" \
  -d '{ "command": "Get-Date -Format yyyy-MM-dd" }'
```

```json
{ "success": true, "stdout": "2026-03-02", "stderr": "", "exitCode": 0 }
```

```bash
# 多行脚本
curl -X POST http://localhost:7439/tools/shell \
  -H "Content-Type: application/json" \
  -d '{
    "command": "$files = Get-ChildItem E:\\docs-service\\docs -Recurse -Filter *.md\n$files | Select-Object Name, Length"
  }'
```

## 二、文档接口 `/open`

### `POST /open/submit` — 提交（新建/覆盖）文档

向 `docs/` 目录写入 Markdown 文档，可指定子目录路径，自动创建不存在的目录。

**请求体**

| 字段        | 类型      | 必填 | 说明                                            |
|-------------|-----------|------|-------------------------------------------------|
| `filename`  | `string`  | ✅   | 文件名或相对路径，如 `guide` / `api/intro`      |
| `content`   | `string`  | ✅   | Markdown 文档正文                               |
| `filetype`  | `string`  | ❌   | 文件类型，默认 `md`（目前仅支持 `md`）          |
| `overwrite` | `boolean` | ❌   | 是否覆盖已有文档，默认 `true`                   |

**响应**

| 字段          | 类型      | 说明                         |
|---------------|-----------|------------------------------|
| `success`     | `boolean` |                              |
| `path`        | `string`  | 实际写入的相对路径           |
| `overwritten` | `boolean` | 是否覆盖了已有文档           |
| `message`     | `string`  | 操作描述                     |
| `html`        | `string`  | Markdown 渲染后的 HTML       |

**示例**

```bash
curl -X POST http://localhost:7439/open/submit \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "notes/今日笔记",
    "content": "# 今日笔记\n\n- 完成了工具文档\n- 修复了 Agent 状态栏"
  }'
```

```json
{ "success": true, "path": "notes/今日笔记.md", "overwritten": false, "message": "文档已创建" }
```

---

### `POST /open/search` — 全文搜索

在 `docs/` 目录下所有 `.md` 文件中进行全文搜索，支持四种匹配模式。

**请求体**

| 字段     | 类型     | 必填 | 说明                                                                 |
|----------|----------|------|----------------------------------------------------------------------|
| `q`      | `string` | ✅   | 搜索词                                                               |
| `mode`   | `string` | ❌   | `fuzzy`（默认）/ `words` / `exact` / `regex`                        |
| `fields` | `string` | ❌   | `both`（默认）/ `title` / `content`                                  |
| `limit`  | `number` | ❌   | 最多返回条数，默认 `20`，最大 `100`                                  |

**匹配模式**

| 模式    | 说明                                              |
|---------|---------------------------------------------------|
| `fuzzy` | 子序列匹配（字符依次出现即命中，类似 fzf）        |
| `words` | 所有关键词均出现（空格分割，顺序无关）            |
| `exact` | 精确子串匹配（忽略大小写）                        |
| `regex` | 正则表达式匹配                                    |

**示例**

```bash
# 模糊搜索
curl -X POST http://localhost:7439/open/search \
  -H "Content-Type: application/json" \
  -d '{ "q": "agent工具" }'

# 正则搜索
curl -X POST http://localhost:7439/open/search \
  -H "Content-Type: application/json" \
  -d '{ "q": "POST /tools/\\w+", "mode": "regex" }'

# 仅搜索标题
curl -X POST http://localhost:7439/open/search \
  -H "Content-Type: application/json" \
  -d '{ "q": "笔记", "fields": "title" }'
```

**响应** `{ success, query, mode, total, results: [{ path, title, score, excerpts }] }`

## 四、Agent 接口 `/agent`

> 基于 GitHub Copilot CLI，需先完成 GitHub 认证（`POST /agent/auth`）。

| 端点                       | 方法 | 说明                                       |
|----------------------------|------|--------------------------------------------|
| `GET /agent/detect`        | GET  | 检测 Copilot CLI 是否可用                  |
| `GET /agent/models`        | GET  | 获取可用模型列表                           |
| `GET /agent/stream`        | GET  | SSE 流（接收 Agent 实时输出）              |
| `POST /agent/start`        | POST | 启动 Agent 任务                            |
| `POST /agent/stop`         | POST | 停止当前 Agent                             |
| `GET /agent/status`        | GET  | 查询当前运行状态                           |
| `GET /agent/history`       | GET  | 获取会话历史列表                           |
| `GET /agent/history-docs`  | GET  | 列出 `docs/history/` 下所有历史文档        |
| `POST /agent/history-docs` | POST | 在 `docs/history/` 新建历史记录文档        |
| `POST /agent/auth`         | POST | 启动 GitHub 认证（SSE 推送授权链接）       |
| `POST /agent/install`      | POST | 安装/更新 Copilot CLI                      |
| `GET /agent/docs`          | GET  | 获取注入给 AI 的系统上下文文档             |

### 启动任务

```bash
curl -X POST http://localhost:7439/agent/start \
  -H "Content-Type: application/json" \
  -d '{
    "task": "分析 docs/guide.md 的内容并给出改进建议",
    "model": "claude-sonnet-4.6",
    "historyDoc": "history/分析记录.md"
  }'
```

**请求体**

| 字段         | 类型     | 必填 | 说明                                           |
|--------------|----------|------|------------------------------------------------|
| `task`       | `string` | ✅   | 要执行的任务描述                               |
| `model`      | `string` | ❌   | 模型名称，默认 `claude-sonnet-4.6`             |
| `historyDoc` | `string` | ❌   | 历史文档路径，输出实时写入                     |
| `saveAs`     | `string` | ❌   | 将完整输出另存为指定文档                       |
| `systemDoc`  | `string` | ❌   | 注入为系统上下文的文档路径                     |

### 接收实时输出（SSE）

```javascript
const es = new EventSource('http://localhost:7439/agent/stream');

es.addEventListener('output',        (e) => console.log(JSON.parse(e.data).text));
es.addEventListener('agent-action',  (e) => console.log('状态:', JSON.parse(e.data).label));
es.addEventListener('done',          (e) => console.log('完成', JSON.parse(e.data)));
es.addEventListener('history-saved', (e) => console.log('已写入:', JSON.parse(e.data).path));
```

**SSE 事件清单**

| 事件名          | 触发时机                          | `data` 字段                             |
|-----------------|-----------------------------------|-----------------------------------------|
| `start`         | Agent 启动时                      | `{ task }`                              |
| `output`        | 每个输出片段                      | `{ text, stream }` (`stdout`/`stderr`)  |
| `agent-action`  | 检测到工具调用/思考/报错等状态    | `{ type, label }`                       |
| `heartbeat`     | 每 3 秒                           | `{ elapsedSec, alive, pid }`            |
| `done`          | 任务结束                          | `{ code, elapsed, task }`               |
| `stopped`       | 手动停止                          | `{ message }`                           |
| `history-saved` | 历史文档写入完成                  | `{ path }`                              |
| `saved`         | saveAs 文档保存完成               | `{ path }`                              |
| `error`         | 发生错误                          | `{ message }`                           |

**`agent-action` type 取值**

| type      | 含义               | 场景                    |
|-----------|--------------------|-------------------------|
| `tool`    | 工具调用中         | CLI 调用 shell/python   |
| `think`   | AI 思考中          | 模型响应延迟            |
| `wait`    | 等待响应           | 网络/API 请求中         |
| `success` | 操作成功           | 工具执行完毕            |
| `failed`  | 操作失败           | 工具报错                |
| `perm`    | 需要权限确认       | CLI 权限提示            |
| `task`    | 执行新子任务       | 任务拆分                |
| `idle`    | 空闲（清空状态栏） | 任务开始/结束时         |

## 五、内置工具 `tools/`

`tools/` 目录下内置以下文件，Agent 调用 shell/python 工具时自动使用，也可手动调用。

### PowerShell 7（本地可移植独立版）

| 项     | 详情                                       |
|--------|--------------------------------------------|
| 版本   | **7.5.4**                                  |
| 可执行 | `tools/pwsh7/pwsh.exe`                     |
| Shim   | `tools/pwsh.cmd`（Agent 调用时自动转发）   |
| 来源   | GitHub PowerShell/PowerShell 官方发布      |

Agent 执行 shell 工具所调用的 `pwsh.exe` 即此本地独立版，**不依赖系统安装**，PowerShell 7 支持跨平台语法、更强的 JSON/REST 处理能力。

**直接使用**

```powershell
# 查看版本
e:\docs-service\tools\pwsh7\pwsh.exe -Command '$PSVersionTable.PSVersion'
# 输出: Major=7  Minor=5  Patch=4

# 执行脚本
e:\docs-service\tools\pwsh7\pwsh.exe -NoProfile -File myScript.ps1

# 单行命令
e:\docs-service\tools\pwsh7\pwsh.exe -Command 'Get-ChildItem | Select Name'
```

**通过 `/tools/shell` 接口调用**（与直接调用等效）

```bash
curl -X POST http://localhost:7439/tools/shell \
  -H "Content-Type: application/json" \
  -d '{ "command": "$PSVersionTable.PSVersion.ToString()" }'
```

```json
{ "success": true, "stdout": "7.5.4", "stderr": "", "exitCode": 0 }
```

**更新到新版本**

```powershell
# 1. 查询最新版本
$latest = (Invoke-RestMethod https://api.github.com/repos/PowerShell/PowerShell/releases/latest).tag_name
Write-Host "最新版本: $latest"

# 2. 下载并替换（在 tools/ 目录下执行）
$ver = $latest.TrimStart('v')
$zip = "ps7-new.zip"
Invoke-WebRequest "https://github.com/PowerShell/PowerShell/releases/download/$latest/PowerShell-$ver-win-x64.zip" -OutFile $zip
Remove-Item .\pwsh7 -Recurse -Force
Expand-Archive $zip -DestinationPath .\pwsh7
Remove-Item $zip
Write-Host "更新完成: $(.\pwsh7\pwsh.exe --version)"
```

### python_runner.py

从 `stdin` 读取 Python 代码并在沙箱中执行，供 `POST /tools/python` 调用。

### shell_runner.ps1

从 `stdin` 读取 PowerShell 命令并执行，输出 JSON，供 `POST /tools/shell` 调用。

### pwsh.cmd（Shim）

将 PATH 中的 `pwsh.exe` 调用转发到 `tools/pwsh7/pwsh.exe`，解决系统未安装 PowerShell 7 的问题。

## 六、COS 图片上传（MagicWorld）

> 适用于 `application/MagicWorld`，服务地址 `http://localhost:8033`。  
> 使用腾讯云对象存储（COS）将本地图标加速分发，无需修改前端代码。

### 前置条件：配置凭证

在 `application/MagicWorld/.env` 中设置以下环境变量，然后重启服务：

```env
SECRET_ID=your_cos_secret_id
SECRET_KEY=your_cos_secret_key
```

| 变量          | 说明                           |
|---------------|--------------------------------|
| `SECRET_ID`   | 腾讯云 COS SecretId            |
| `SECRET_KEY`  | 腾讯云 COS SecretKey           |

COS 配置：
- **Bucket**：`magicworld-1304036735`
- **Region**：`ap-guangzhou`
- **图标前缀**：`gameicons/`
- **公网访问地址**：`https://magicworld-1304036735.cos.ap-guangzhou.myqcloud.com/gameicons/<文件名>`

---

### `GET /api/cos/status` — 检查 COS 凭证状态

```bash
curl http://localhost:8033/api/cos/status
```

```json
{
  "configured": true,
  "bucket": "magicworld-1304036735",
  "region": "ap-guangzhou",
  "hint": null
}
```

未配置凭证时 `configured` 为 `false`，`hint` 提示重启服务。

---

### `POST /api/cos/upload-test` — 测试上传（前 N 张）

上传前 N 张图标到 COS，用于验证凭证和连通性。

**请求体**

| 字段     | 类型     | 必填 | 说明                         |
|----------|----------|------|------------------------------|
| `limit`  | `number` | ❌   | 上传数量，默认 `5`，最大 `20` |
| `prefix` | `string` | ❌   | COS 路径前缀，默认 `gameicons/` |

```bash
curl -X POST http://localhost:8033/api/cos/upload-test \
  -H "Content-Type: application/json" \
  -d '{ "limit": 3 }'
```

```json
{
  "summary": "上传 3 成功，0 失败",
  "results": [
    { "file": "skillicon_1.png", "cosKey": "gameicons/skillicon_1.png", "url": "https://...", "ok": true }
  ]
}
```

---

### `POST /api/cos/upload-all` — 批量上传全部图标

将 `public/icons/` 目录下所有 `.png` 文件批量上传到 COS。

**请求体**

| 字段     | 类型     | 必填 | 说明                           |
|----------|----------|------|--------------------------------|

```bash

```


> **注意**：上传约 110 张图标需 1–3 分钟，网络波动导致的少量失败可重复调用，已上传的文件会被覆盖（幂等）。

### 完整上传流程

```bash
# 1. 检查 COS 是否配置
curl http://localhost:8033/api/cos/status

# 2. 测试上传 3 张（验证凭证）
curl -X POST http://localhost:8033/api/cos/upload-test \
  -H "Content-Type: application/json" \
  -d '{ "limit": 3 }'

# 3. 批量上传全部图标
curl -X POST http://localhost:8033/api/cos/upload-all \
  -H "Content-Type: application/json" \
  -d '{}'

# 4. 验证 dict API 返回 COS URL
curl http://localhost:8033/api/dict/skills | python -m json.tool | grep icon | head -3
```
