# 回复：哪种记忆策略适合长期重复工作场景

**时间**: 2026-03-12  
**针对问题**: 每个 session 长期执行某类工作，几十轮迭代后仍能快速记忆怎么做，不需要每次重新查阅

---

## 结论：推荐「动态 SOP 文档」策略（程序性记忆模式）

你描述的场景本质是**程序性记忆（Procedural Memory）**——记住"怎么做"而非"做了什么"。

最适合的策略是：**Reflexion + 自更新 SOP 文档**

---

## 具体实现方案

### 核心思路

每次任务完成后，Agent 不只是归档结果，而是**更新一份活的经验文档**：

`
第1轮执行 → 完成任务 → 反思总结 → 写入/更新 SOP.md
第2轮执行 → 读取 SOP.md → 执行任务 → 更新 SOP.md（如有新发现）
...
第N轮执行 → SOP.md 已非常完善 → 几乎不需要修改，直接复用
`

### 文档结构示例

`
docs/agent/sop/{任务类型}.md
├── ## 步骤清单（每次执行前读取）
├── ## 常见错误与解决方案（迭代积累）
├── ## 最优工具/API 选择（经验沉淀）
└── ## 版本历史（追踪改进轨迹）
`

### 与现有系统结合

在任务完成后，在 waitprocess 轮询前，增加一步：

`powershell
# 读取当前任务类型的 SOP
 = "E:\docs-service\docs\agent\sop\{任务类型}.md"
 = Get-Content  -Raw -ErrorAction SilentlyContinue

# 执行任务时将 SOP 注入上下文
# 任务完成后更新 SOP
`

---

## 为什么不选其他方案

| 方案 | 不适合的原因 |
|------|------------|
| 向量数据库(RAG) | 适合检索"做了什么"，不适合记住"怎么做" |
| 对话摘要 | 压缩的是对话内容，不是操作程序 |
| 知识图谱 | 过于复杂，适合实体关系，不适合流程记忆 |
| 全量历史 | 几十轮后 context 爆炸，且噪声多 |

---

## 实施建议

1. **立即可用**：为每种任务类型在 docs/agent/sop/ 下建一个 .md 文件
2. **每次任务后**：让 Agent 判断是否需要更新 SOP（5-10行简洁经验）
3. **每次任务前**：优先读取对应 SOP，而不是从零推导
4. **SOP 精炼**：迭代 10 轮后做一次"蒸馏"，去掉冗余，保留精华

这样 30 轮后，SOP 文档会非常稳定和精炼，Agent 每次只需读这一份文档就能快速上手。
