# Agent 长记忆功能实现方案

> 整理时间：2026-03-12  
> 来源：Lilian Weng Blog、Pinecone、学术论文

---

## 一、记忆体系概述

类比人类大脑，Agent 的记忆可分为三层：

| 层级 | 类比 | 技术实现 |
|------|------|----------|
| 感知记忆 | Sensory Memory | 输入 Embedding 表示（文本/图像） |
| 短期记忆 | Short-Term Memory | In-Context（上下文窗口，有限长度） |
| 长期记忆 | Long-Term Memory | **外部向量存储 + 快速检索** |

---

## 二、长记忆核心实现方案

### 方案 1：向量数据库（RAG）⭐ 主流

**原理**：将历史对话/知识片段 Embedding 存入向量库，查询时通过语义相似度检索相关记忆注入上下文。

**实现流程**：
`
用户输入 → Embedding模型 → 向量检索(ANN) → Top-K记忆 → 拼入Prompt → LLM生成
`

**常用向量数据库**：
- Chroma：轻量本地部署，适合小规模
- Pinecone：云端托管，高性能
- Weaviate：开源全功能
- FAISS（Facebook）：纯向量索引库

**ANN 检索算法**：
- HNSW：层次小世界图，速度最快，推荐
- FAISS：高维空间聚类量化，大规模场景
- LSH：局部敏感哈希，极简实现
- ScaNN（Google）：各向异性量化，精度最高

---

### 方案 2：对话缓冲与摘要记忆（LangChain）

| 类型 | 说明 | 适用 |
|------|------|------|
| ConversationBufferMemory | 原始对话全量保存 | 短对话 |
| ConversationSummaryMemory | LLM自动摘要压缩 | 长对话 |
| ConversationBufferWindowMemory | 滑动窗口保留最近K轮 | 平衡方案 |
| VectorStoreRetrieverMemory | 向量检索历史 | 语义检索 |

---

### 方案 3：Reflexion 框架（自反思记忆）

**原理**：Agent 执行任务后对失败轨迹进行自我反思，将反思结果存入工作记忆，下次执行时参考。

流程：执行任务 → 评估结果 → 生成自我反思 → 存入记忆 → 下次任务注入最近3条反思

特点：无需梯度更新，纯提示工程实现，特别适合需要多次试错的推理任务。

---

### 方案 4：结构化外部存储

| 存储类型 | 工具 | 适用场景 |
|---------|------|---------|
| Key-Value | Redis / SQLite | 用户配置、状态持久化 |
| 关系数据库 | PostgreSQL | 结构化事实记忆 |
| 知识图谱 | Neo4j | 实体关系记忆 |
| 文件系统 | Markdown文件 | 本系统当前方案 |

---

### 方案 5：MemGPT / 内存管理 Agent

模拟操作系统内存管理：
- 主上下文（in-context）= 工作内存（快速访问）
- 外部存储（out-of-context）= 长期存储（按需换入）

核心操作：recall_memory() / archive_memory() / search_memory()

## 三、方案对比

| 方案 | 复杂度 | 语义理解 | 精确查找 | 推荐场景 |
|------|--------|---------|---------|---------|
| 向量数据库(RAG) | 中 | 极高 | 中 | 知识库、长期对话 |
| 对话摘要 | 低 | 中 | 低 | 聊天机器人 |
| 文件系统(.md) | 极低 | 低 | 中 | 本系统当前方案 |
| Reflexion | 中 | 高 | 低 | 推理/规划任务 |
| 知识图谱 | 高 | 中 | 极高 | 复杂关系推理 |

## 四、本系统（爱丽丝）现状与升级建议

当前实现：Markdown 文件系统作为长记忆存储

`
docs/agent/chat/{sessionId}/
├── waitprocess/    ← 待处理任务队列
├── hasprocess/     ← 已处理任务归档
└── history/        ← 会话历史记忆文件
`

优点：零依赖、人类可读、版本可控
局限：无语义检索、记忆量大后查找困难

升级建议：引入本地 Chroma 向量数据库对 docs/ 目录建立语义索引，实现语义增强搜索。
