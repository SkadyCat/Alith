# 文生图→3D 流程分析与策略

> 分析时间: 2026-03-04

---

## 一、根本原因：为什么模型和图完全不像？

### 问题链路

```
用户期望：   文本 → 生成图 ──[图转3D]──→ 与图匹配的3D模型
实际发生：   文本 → 生成图     X          程序化几何体 (固定顶点/三角面)
```

### 具体原因

1. **云端 API 未开通**  
   腾讯云混元3D API (`ai3d.tencentcloudapi.com`) 返回：  
   `{"Code":"ResourceUnavailable.NotExist","Message":"计费状态未知，请确认是否已在控制台开通服务"}`  
   → 上次和现在都无法使用。

2. **Fallback 是程序化几何，与图完全无关**  
   上次 session 的"龙模型"是手写代码直接拼 GLB 顶点数据（body/neck/head/wings/tail 等固定几何），  
   **根本没有读取或分析 ComfyUI 生成的图片**，所以当然不像。

---

## 二、正确的技术路径：图像转3D (Image-to-3D)

真正的图→3D 需要一个「图像到3D重建模型」。目前主流方案：

| 方案 | 类型 | VRAM需求 | 速度 | 质量 |
|------|------|----------|------|------|
| **TripoSR** | 本地开源 | ~4-6GB | 5-10秒 | ⭐⭐⭐ |
| **Stable Fast 3D** | 本地开源 | ~7-8GB | 10-20秒 | ⭐⭐⭐⭐ |
| **Hunyuan3D-2 本地** | 本地开源 | ~12GB+ | 30-60秒 | ⭐⭐⭐⭐⭐ |
| 混元3D 云API | 云端付费 | 无需 | 30-120秒 | ⭐⭐⭐⭐⭐ |
| Tripo3D API | 云端付费 | 无需 | 10-30秒 | ⭐⭐⭐⭐ |

## 四、推荐策略

### 首选方案：本地 TripoSR（速度快，资源消耗小）

**完整流程：**
```
1. ComfyUI (Flux) 文生图 → 输出 PNG
2. TripoSR 图转3D → 输出 mesh.obj / mesh.glb (有纹理)
3. 上传 COS → 返回 URL
```

**安装步骤：**
```bash
# 在 Flux venv 中安装
cd E:\AIGC\Flux\backend
venv\Scripts\pip install triposg  # 或 pip install git+https://github.com/VAST-AI-Research/TripoSR
# 下载模型 (HuggingFace: stabilityai/TripoSR)
# 约 1.4GB 模型文件
```

**推理代码思路：**
```python
from tsr.system import TSR
model = TSR.from_pretrained("stabilityai/TripoSR", ...)
scene_codes = model([image], device="cuda")
meshes = model.extract_mesh(scene_codes)
meshes[0].export("output.glb")
```

### 备选方案：Stable Fast 3D

HuggingFace: `stabilityai/stable-fast-3d`  
质量更好，但需要约 7-8GB VRAM。

### 次选方案：开通腾讯云混元3D API

如果要使用云端：
- 进入腾讯云控制台 → AI 3D → 开通计费
- 模型选 `hunyuan-3d-2.0` 或 `hunyuan-3d-turbo`
- MagicWorld 的 `/api/3d/generate` 接口已写好，开通即可用
