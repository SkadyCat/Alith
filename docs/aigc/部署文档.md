# Flux Image Generator 部署文档

> 基于 Black Forest Labs FLUX.1 模型的本地图像生成系统，使用 Gradio 构建 Web UI，支持 ModelScope 国内直连下载，无需科学上网。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11（已验证）、Linux |
| Python | 3.10.x |
| GPU | NVIDIA RTX 系列（推荐 8GB+ VRAM） |
| CUDA | 12.x（驱动版本 ≥ 520） |
| 磁盘空间 | ≥ 50 GB（模型约 33 GB） |

---

## 二、项目结构

```
flux-image-generator/
├── app.py            # Gradio Web UI 主入口
├── config.py         # 模型注册表与全局配置
├── model_manager.py  # 模型加载、推理、进度追踪
├── requirements.txt  # Python 依赖清单
├── runserver.bat     # 一键启动脚本（Windows）
└── outputs/          # 生成图像保存目录
```

## 三、安装步骤

### 1. 安装 Python 依赖

**核心依赖（需先单独安装 CUDA 版 PyTorch）：**

```bash
# 安装 CUDA 12.4 版 PyTorch（RTX 20/30/40 系列通用）
pip install torch==2.5.1+cu124 torchvision==0.20.1+cu124 \
    --index-url https://download.pytorch.org/whl/cu124

# 安装其余依赖（使用阿里云镜像加速）
pip install diffusers transformers accelerate huggingface_hub \
    sentencepiece protobuf Pillow gradio==5.23.3 \
    omegaconf safetensors optimum modelscope \
    -i https://mirrors.aliyun.com/pypi/simple/
```

**注意事项：**
- `gradio` 需指定 `5.23.3`，6.x 版本 API 有重大不兼容变更，4.44.x 与 huggingface_hub 1.x 不兼容
- `modelscope` 用于国内直连下载模型，避免访问 HuggingFace 被封

### 2. 验证安装

```bash
python -c "import torch; print('CUDA:', torch.cuda.is_available()); print('GPU:', torch.cuda.get_device_name(0))"
```

## 四、模型配置

系统在 `config.py` 中注册了四个模型选项：

| 模型名 | 来源 | 需要 Token | 推理步数 | 适用场景 |
|--------|------|:----------:|:-------:|---------|
| FLUX.1-schnell [ModelScope★] | ModelScope | ❌ | 4 步 | 国内推荐，快速出图 |
| FLUX.1-dev [ModelScope] | ModelScope | ❌ | 28 步 | 国内推荐，高质量 |
| FLUX.1-schnell [HuggingFace] | HuggingFace | ✅ | 4 步 | 需接受许可协议 |
| FLUX.1-dev [HuggingFace] | HuggingFace | ✅ | 28 步 | 需接受许可协议 |

**显存优化配置（`config.py`）：**

```python
ENABLE_CPU_OFFLOAD = True          # RTX 4060 8GB 建议开启
ENABLE_SEQUENTIAL_OFFLOAD = False  # VRAM < 6GB 时改为 True
TORCH_DTYPE = "bfloat16"           # 减少显存占用
```

**各显存方案参考：**

| 显存 | 推荐配置 |
|------|---------|
| ≥ 16 GB | 关闭所有 Offload |
| 8 GB | 开启 `ENABLE_CPU_OFFLOAD` |
| 4–6 GB | 开启 `ENABLE_SEQUENTIAL_OFFLOAD` |
| 无 GPU | `DEVICE = "cpu"` （极慢，仅用于测试） |

## 五、启动服务

### Windows 一键启动

双击 `runserver.bat`，脚本会自动：
1. 检测 Python 环境
2. 释放被占用的 7860 端口
3. 设置 `HF_ENDPOINT=https://hf-mirror.com`（HuggingFace 国内镜像）
4. 启动 Gradio Web UI

### 手动启动

```bash
cd flux-image-generator

# Windows 设置环境变量
set HF_ENDPOINT=https://hf-mirror.com

python app.py
# 可选参数：
# --port 8080       修改端口
# --share           生成公网分享链接
# --model "FLUX.1-schnell [ModelScope★推荐]"  启动时预加载模型
```

服务启动后访问：**http://localhost:7860**

## 六、使用流程

1. 打开 **http://localhost:7860**
2. 左侧「模型配置」区域选择 `FLUX.1-schnell [ModelScope★推荐]`
3. 点击 **⬇️ 加载模型**
   - 首次加载会从 ModelScope 下载约 **33 GB** 文件，需要时间
   - 「下载/加载进度」框实时滚动显示每个文件的下载速度与进度
   - 下载完成后，后续加载直接从本地缓存读取，秒速完成
4. 在右侧输入英文提示词（Flux 对英文理解更准确）
5. 点击 **🎨 生成图像**

**提示词技巧：**
- 具体描述场景、光线、风格，例如：`A misty mountain valley at sunrise, photorealistic, 8K, cinematic lighting`
- 加质量词：`best quality, highly detailed, sharp focus`
- 加风格词：`oil painting, watercolor, illustration, cinematic`

## 七、常见问题排查

### Q: 加载失败 `403 Forbidden`
**原因：** FLUX 模型在 HuggingFace 是 Gated 仓库，即使 schnell 也需要接受许可协议才能访问。  
**解决：** 使用 `ModelScope` 源（UI 中默认选项），无需 Token，国内直连。

### Q: 加载失败 `404 Not Found`
**原因：** ModelScope 模型 ID 大小写错误。  
**正确 ID：**
- `AI-ModelScope/FLUX.1-schnell`
- `AI-ModelScope/FLUX.1-dev`

### Q: GPU 显存不足（OOM）
**解决：**
1. 在 `config.py` 中设置 `ENABLE_CPU_OFFLOAD = True`
2. 降低生成分辨率（如 512×512）
3. VRAM < 6GB 时改为 `ENABLE_SEQUENTIAL_OFFLOAD = True`

### Q: HuggingFace 下载超慢或无法连接
**解决：** 已内置 hf-mirror 镜像，启动时设置：
```bash
set HF_ENDPOINT=https://hf-mirror.com
```

### Q: `ImportError: cannot import name 'HfFolder'`
**原因：** Gradio 4.44.x 与 huggingface_hub 1.x 不兼容。  
**解决：** 指定安装 `gradio==5.23.3`

## 八、技术架构

```
app.py (Gradio UI)
  │
  ├── model_manager.py
  │     ├── ModelScope snapshot_download  ← 国内直连下载
  │     ├── HuggingFace FluxPipeline      ← hf-mirror 加速
  │     ├── _TqdmCapture (stderr 拦截)    ← 实时进度捕获
  │     └── FluxPipeline (diffusers)      ← 推理引擎
  │
  └── config.py
        ├── 模型注册表（4个模型选项）
        └── 硬件/显存优化配置
```
