# 爱丽丝分身部署文档

> 本文档描述如何将爱丽丝（DocSpace + Agent 服务）完整部署到新机器。  
> **不包含** `application/` 目录下的应用（MagicWorld 等），如需请单独部署。

---

## 一、爱丽丝是什么

爱丽丝是一个以 **DocSpace** 为核心的本地 AI Agent 服务系统，运行于 `http://localhost:7439`，具备以下能力：

| 能力模块 | 说明 |
|----------|------|
| 📄 **Markdown 文档站** | 网页端在线编辑、浏览、全文搜索 Markdown 文档 |
| 🔌 **对外开放接口 `/open`** | Agent / 外部系统可通过 HTTP 提交、搜索、读取文档 |
| 🤖 **AI Agent `/agent`** | 基于 GitHub Copilot CLI，接受任务、实时 SSE 输出、保存历史 |
| 🐍 **Python 执行 `/tools/python`** | 服务端沙箱执行任意 Python 代码片段 |
| 💻 **Shell 执行 `/tools/shell`** | 服务端执行 PowerShell 7 命令/脚本 |
| 📁 **文件管理 `/api`** | 创建/读取/保存/删除/上传文档和目录 |
| 📡 **文件实时推送 `/api/watch`** | SSE 文件变更监听，网页自动刷新 |
| 🔄 **会话管理 `/api/dialogue`** | 保存/加载 Agent 对话会话配置 |
| 🔁 **重启接口 `/api/restart`** | 触发服务自重启（通过 rundoc.bat） |

## 二、前置依赖

| 依赖 | 版本要求 | 安装方式 |
|------|----------|----------|
| **Node.js** | v18+ | https://nodejs.org |
| **Python** | 3.8+ | https://python.org |
| **GitHub CLI** | 最新版 | https://cli.github.com |
| **GitHub Copilot CLI 扩展** | — | 见下方步骤 |
| **PowerShell 7** | 7.x（可移植版） | 见下方步骤 |

### 安装 GitHub Copilot CLI（Agent 核心）

```bash
# 1. 安装 GitHub CLI
winget install GitHub.cli

# 2. 登录 GitHub
gh auth login

# 3. 安装 Copilot CLI 扩展
gh extension install github/gh-copilot

# 4. 验证
gh copilot --version
```

### 安装 PowerShell 7（可移植版，放到 tools/pwsh7/）

在项目根目录 PowerShell 中执行（需要网络）：

```powershell
$ver = "7.5.4"
$zip = "ps7.zip"
# 可使用 clash 代理加速: $env:HTTP_PROXY="http://127.0.0.1:7890"
Invoke-WebRequest "https://github.com/PowerShell/PowerShell/releases/download/v$ver/PowerShell-$ver-win-x64.zip" -OutFile $zip
New-Item -ItemType Directory -Force -Path "tools\pwsh7"
Expand-Archive $zip -DestinationPath "tools\pwsh7" -Force
Remove-Item $zip
.\tools\pwsh7\pwsh.exe --version  # 应输出 PowerShell 7.5.4
```

## 三、部署步骤

### 1. 解压 zip 包

将 `alith-core.zip` 解压到目标目录，例如 `D:\alith`：

```
D:\alith\
  server.js
  package.json
  routes\
  public\
  tools\          ← python_runner.py, shell_runner.ps1, pwsh.cmd
  docs\           ← 所有知识库文档
  data\
```

### 2. 安装 Node 依赖

```bash
cd D:\alith
npm install
```

安装的包（见 package.json）：
- `express` — Web 框架
- `marked` — Markdown 渲染
- `highlight.js` — 代码高亮
- `chokidar` — 文件监听

### 3. 安装 PowerShell 7

按上方「安装 PowerShell 7」步骤，在 `D:\alith\tools\pwsh7\` 目录下放好 `pwsh.exe`。

### 4. 修改系统配置文档

编辑 `docs\agent\application.md`，将路径改为实际部署路径：

```markdown
# 本系统名称 "爱丽丝"
爱丽丝所在的工作路径 在 D:\alith

# 在哪可以创建应用?
D:\alith\application

# 如何使用工具，从哪里去查找资源
http://localhost:7439/open/study

# 如何重启 爱丽丝
运行 rundoc.bat。
```

### 5. 创建启动脚本 `rundoc.bat`

在项目根目录创建 `rundoc.bat`（已在 zip 中提供模板，修改路径后使用）：

```bat
@echo off
cd /d %~dp0
echo 正在启动爱丽丝...
echo 访问地址: http://localhost:7439
node server.js
pause
```

### 6. 启动服务

双击 `rundoc.bat` 或在命令行运行：

```bat
rundoc.bat
```

访问 `http://localhost:7439` 即可看到文档站界面。

## 四、GitHub 认证（Agent 功能）

Agent 功能需要 GitHub Copilot 授权，首次使用需认证：

```bash
# 方式一：通过接口触发（SSE 推送设备码）
curl -X POST http://localhost:7439/agent/auth

# 方式二：命令行直接登录（推荐）
gh auth login
```

认证后，Agent 即可通过 `/agent/start` 接受任务，通过 `/agent/stream` SSE 实时输出结果。

## 七、常见问题

**Q: Agent 无法启动，提示 Copilot CLI 不可用？**  
A: 运行 `gh copilot --version` 确认扩展已安装，再运行 `gh auth login` 确认已登录。

**Q: PowerShell 执行工具报错？**  
A: 确认 `tools\pwsh7\pwsh.exe` 存在。如不存在，按第二节步骤下载安装。

**Q: 端口 7439 被占用？**  
A: 找到占用进程并终止，然后重启：
```powershell
Get-Process -Id (Get-NetTCPConnection -LocalPort 7439).OwningProcess | Stop-Process -Force
node server.js
```

**Q: 如何更改端口？**  
A: 修改 `server.js` 顶部 `const PORT = 7439;`。

**Q: 如何让爱丽丝知道当前项目上下文？**  
A: 编辑 `docs\agent\application.md`，在其中描述系统名称、工作路径、特殊工具等信息。Agent 启动任务时会自动注入该文档作为系统上下文。

**Q: docs/ 里的文档会丢失吗？**  
A: 不会，docs/ 是持久化目录，服务重启不会清空。建议定期备份。
