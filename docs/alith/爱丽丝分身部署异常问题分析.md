# 爱丽丝分身部署异常问题分析

> 记录时间：2026-03-03  
> 部署路径：`E:\Alith`  
> 部署人：GitHub Copilot（AI Agent）

---

## 一、package.json JSON 格式错误

**现象**：`npm install` 报错 `Expected double-quoted property name in JSON`

**原因**：`package.json` 的 `scripts` 字段末尾存在尾随逗号，不符合 JSON 规范。

```json
// 错误写法
"scripts": {
  "start": "node server.js",   ← 尾随逗号
},
```

**修复**：移除尾随逗号。

---

## 二、GitHub CLI (`gh`) 未安装

**现象**：点击「GitHub 认证」无响应，`spawnAuth()` 启动失败。

**原因**：新机器未安装 GitHub CLI，`gh auth login` 无法执行。

**修复**：
```powershell
winget install GitHub.cli --silent --accept-package-agreements --accept-source-agreements
```

---

## 三、`spawnAuth()` 调用命令错误

**现象**：点击认证按钮无输出，AuthProcess 启动后立即退出。

**原因**：原代码调用 `copilotCmd login`（依赖 `@github/copilot` npm 包），但该包未安装，且认证应使用 `gh auth login`。

**修复**：新增 `findGhCmd()` 函数定位 `gh.exe`，`spawnAuth()` 改为调用 `gh auth login`，并自动通过 stdin 回答交互式提示。

---

## 四、`@github/copilot` npm 包未安装

**现象**：`/agent/detect` 返回 `available: false`，界面提示"Copilot CLI 未安装"。

**原因**：`Agent` 的 autopilot 模式基于 `@github/copilot` npm 包（非 `gh copilot` 扩展），需单独安装。

**修复**：
```powershell
npm install -g @github/copilot
```

安装后版本：`GitHub Copilot CLI 0.0.420`

---

## 五、`detectCopilotCmd()` 误返回 `gh copilot`

**现象**：安装 `gh` 后，Agent 仍然失败（exit code 1），错误日志无实质内容。

**原因**：修复 `gh auth` 时顺带把 `detectCopilotCmd()` 改为返回 `gh copilot`，但 `gh copilot` 扩展没有 `--autopilot` 模式，不是 Agent 所需组件。

**修复**：恢复 `detectCopilotCmd()` 只检测 `@github/copilot` npm 包路径（node-script 模式）。

---

## 六、GH_TOKEN 未注入子进程环境

**现象**：`gh auth login` 认证成功，但启动 Agent 仍报：
```
Error: No authentication information found.
```

**原因**：`@github/copilot` 作为独立子进程运行，不继承宿主进程的 `gh` 会话凭证；需通过环境变量 `COPILOT_GITHUB_TOKEN` / `GH_TOKEN` / `GITHUB_TOKEN` 显式传入。

**修复**：在 `buildEnv()` 中新增 `getGhToken()` 调用，每次构建子进程环境时动态执行 `gh auth token` 获取新鲜 token，并同时注入三个变量：
```javascript
env.GH_TOKEN             = ghToken;
env.COPILOT_GITHUB_TOKEN = ghToken;  // autopilot 优先读此变量
env.GITHUB_TOKEN         = ghToken;
```

---

## 七、GitHub CLI 不在子进程 PATH 中

**现象**：注入 `GH_TOKEN` 后仍报认证失败（autopilot 特有）。

**原因**：`@github/copilot` autopilot 模式内部会调用 `gh` 命令（用于获取 Copilot 授权），而子进程的 `PATH` 来自服务器启动时的快照，不包含 GitHub CLI 安装目录（`C:\Program Files\GitHub CLI`）。

**修复**：`buildEnv()` 中将 GitHub CLI 目录明确注入子进程 PATH：
```javascript
const ghCliDir = path.dirname(findGhCmd());  // C:\Program Files\GitHub CLI
env.PATH = [pwsh7Dir, toolsDir, ghCliDir, npmBinDir, env.PATH].join(pathSep);
```

---

## 八、代理强制注入导致网络超时风险

**现象**：Clash 未开启时，代理行为不一致，存在超时隐患。

**原因**：`buildEnv()` 无条件设置 `HTTPS_PROXY=http://127.0.0.1:7890`，Clash 未开时请求会失败。

**修复**：启动前用 `netstat -an` 检测 7890 端口是否监听，仅在确认开放时才注入代理变量。

---

## 九、SSE 缓冲区 replay 旧错误

**现象**：服务重启后，前端界面仍显示旧的认证失败错误。

**原因**：`agentBuffer` 缓存上一次任务的所有 SSE 事件，新客户端连接时会 replay 全部历史。重启前的错误事件也被重放给了用户。

**影响**：误导用户认为修复未生效（实际服务器已正常）。

**建议**：服务重启时清空 `agentBuffer`（目前已在每次新任务开始时清空，但旧错误事件仍残留直到下一次任务）。

---

## 十、模型列表过时

**现象**：选择 `GPT-4.1` 等模型时，CLI 报 `argument is invalid`。

**原因**：`MODEL_LIST` 中的模型 ID 未跟随 Copilot CLI 版本更新。

**修复**：同步更新为 v0.0.420 实际支持的模型：
- `claude-sonnet-4.6` / `claude-opus-4.6` / `claude-opus-4.6-fast` / `claude-haiku-4.5`
- `gpt-5.2` / `gpt-5.2-codex` / `gpt-5.3-codex`
- `gemini-3-pro-preview`
