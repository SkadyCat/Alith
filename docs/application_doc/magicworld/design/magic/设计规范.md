# MagicWorld 魔法系统设计规范

> 基于 JSON **组合**模式实现的魔法系统，无需继承即可实现极高灵活性。

---

## 核心设计哲学

JSON 不支持原生继承，因此系统采用**组合（Composition）**代替继承，以三层结构定义技能：

```
技能实例 (spell)
  └─ delivery.type      →  投射类型（projectile/area/persistent/instant/beam/summon）
       └─ delivery.modifiers[]  →  修饰符列表（split/bounce/chain/aura/...）
  └─ on_hit[]           →  命中效果（damage/dot/cc/buff/debuff/...）
  └─ element            →  元素类型（fire/ice/lightning/...）
```

---

## 数据文件结构

| 文件 | 作用 |
|------|------|
| `elements.json` | 9种元素：属性、弱点、DoT类型 |
| `delivery_types.json` | 6种投射类型：参数定义 + 兼容修饰符列表 |
| `modifiers.json` | 19种修饰符：分6个类别，含冲突规则 |
| `effects.json` | 10种效果：damage/dot/cc/buff/debuff/heal等 |
| `spells.json` | 技能实例：组合以上三层的具体技能 |

---

## 修饰符类别

| 类别 | 说明 | 示例 |
|------|------|------|
| `trajectory` | 改变弹体轨迹 | split分裂、bounce弹跳、pierce穿透、homing追踪 |
| `area` | 改变范围逻辑 | expand扩大、linger残留、cascade级联 |
| `persistent` | 改变持续行为 | pulse脉冲、aura光环、rotating旋转 |
| `trigger` | 触发额外事件 | nova_on_expire爆散、echo回响 |
| `amplifier` | 全局强化 | empowered |
| `tempo` | 改变时序节奏 | barrage连射、slow_projectile减速弹 |

---

## 兼容性规则（JSON 实现的约束系统）

每个修饰符包含：
- `applies_to[]` — 可用于哪些投射类型
- `conflicts[]` — 与哪些修饰符互斥

每个投射类型包含：
- `compatible_modifiers[]` — 白名单列表

API `POST /api/magic/validate` 实时校验技能实例的合法性。

## 示例技能：奥术镜（Arcane Mirror）

```json
{
  "delivery": {
    "type": "projectile",
    "modifiers": [
      { "id": "split",  "params": { "count": 4, "trigger": "on_hit", "inherit_mods": true } },
      { "id": "bounce", "params": { "count": 2, "damage_decay": 0.8 } },
      { "id": "echo",   "params": { "delay": 0.4, "damage_mult": 0.4 } }
    ]
  },
  "on_hit": [
    { "effect": "damage", "params": { "base": "100%", "crit_chance": 0.2 } },
    { "effect": "mark",   "params": { "trigger_on": "crit", "effect_id": "arcane_burst" } }
  ]
}
```

效果：发射→命中→分裂4枚→每枚弹跳2次→0.4s后回响一次，暴击时触发奥术爆发标记。

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/magic/elements` | 元素列表 |
| GET | `/api/magic/delivery-types` | 投射类型列表 |
| GET | `/api/magic/modifiers` | 修饰符列表 |
| GET | `/api/magic/effects` | 效果列表 |
| GET | `/api/magic/spells` | 技能实例列表 |
| POST | `/api/magic/validate` | 校验技能兼容性 |
