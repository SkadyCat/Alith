# 提示词生成规范

> **路径**: `docs/keywords/`
> **用途**: 为 3D 模型生成应用提供可选的提示词库

---

## 一、提示词条目结构

每条提示词包含三个核心要素：

| 字段 | 说明 | 示例 |
|------|------|------|
| **主题** | 怪物/角色的名称，用 `## 标题` 表示 | `## 骷髅战士` |
| **提示词** | 英文提示词内容，用于 3D 生成 | `a skeletal warrior with glowing red eyes...` |
| **中文说明** | 提示词的中文解释，方便理解和筛选 | `一个有着发光红眼的骷髅战士...` |

---

## 二、Markdown 文件格式规范

```markdown
# 文件标题

## 主题名称

- **提示词**: [英文提示词，建议 20~80 词，包含: 外观/风格/材质/光影/渲染风格]
- **中文说明**: [中文简要说明，20~60 字]

## 另一个主题

- **提示词**: ...
- **中文说明**: ...
```

### 提示词书写建议

1. **主体描述** — 怪物类型、体型、特征（如 `skeletal warrior`, `giant spider demon`）
2. **外观细节** — 颜色、材质、装备（如 `dark plate armor`, `glowing red eyes`）
3. **风格标签** — 艺术风格（如 `dark fantasy`, `gothic`, `horror style`, `3D render`）
4. **渲染描述** — 渲染质量和用途（如 `high quality 3D model`, `game character`, `concept art`）

---

## 三、提示词 .md 转 JSON 规范

### 解析规则

1. 以 `## ` 开头的行作为**主题分隔符**，提取主题名称
2. 以 `- **提示词**:` 开头的行提取英文提示词内容
3. 以 `- **中文说明**:` 开头的行提取中文说明内容

### JSON 输出格式

```json
{
  "file": "暗黑怪物提示词.md",
  "prompts": [
    {
      "theme": "骷髅战士",
      "prompt": "a skeletal warrior with glowing red eyes, dark plate armor...",
      "description": "一个有着发光红眼的骷髅战士，暗色板甲"
    }
  ]
}
```

### 解析示例（JavaScript）

```javascript
function parseMdPrompts(content, filename) {
  const prompts = [];
  const sections = content.split(/^## /m).slice(1);
  for (const section of sections) {
    const lines = section.split('\n');
    const theme = lines[0].trim();
    const promptMatch = section.match(/\*\*提示词\*\*:\s*(.+)/);
    const descMatch = section.match(/\*\*中文说明\*\*:\s*(.+)/);
    if (theme && promptMatch) {
      prompts.push({
        theme,
        prompt: promptMatch[1].trim(),
        description: descMatch ? descMatch[1].trim() : ''
      });
    }
  }
  return { file: filename, prompts };
}
```

## 四、文件命名约定

| 文件名 | 说明 |
|--------|------|
| `提示词生成规范.md` | 本规范文件（不作为提示词库） |
| `暗黑怪物提示词.md` | 暗黑 like 风格怪物提示词（100条） |
| `*.md` | 其他主题提示词集合 |

> 除 `提示词生成规范.md` 外，所有 `.md` 文件均会被加载为提示词库。
