# 如何使用 DocSpace

DocSpace 是一个基于 Markdown 的轻量文档服务，支持网页浏览、在线编辑和外部接口提交。

---

## 一、网页端使用

访问 `http://localhost:7439` 即可打开网页。

- **左侧** 是文件树，点击文档名称即可打开
- **中间左侧** 是 Markdown 编辑器，直接输入内容
- **中间右侧** 是实时预览，输入后 300ms 自动刷新
- **顶部** 保存按钮 或快捷键 `Ctrl+S` 保存文档
- 右键文件名可**删除**文档
- 点击 **+** 图标可新建文档（支持子目录，如 `notes/daily`）
- 顶部搜索框支持**文件名模糊过滤**

## 二、对外接口（/open）

所有接口均返回 JSON，格式为：

```json
{ "success": true,  "...": "..." }
{ "success": false, "error": "错误描述" }
```

### 1. 提交文档 — `POST /open/submit`

新建或覆盖一篇文档。

**请求 Body（JSON）：**

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|:---:|:---:|---|
| `filename` | string | ✅ | — | 文件名或路径，如 `guide`、`api/intro` |
| `content` | string | ✅ | — | Markdown 正文 |
| `filetype` | string | — | `md` | 文件类型，当前仅支持 `md` |
| `overwrite` | boolean | — | `true` | 是否覆盖同名文档 |

**示例：**

```bash
curl -X POST http://localhost:7439/open/submit \
  -H "Content-Type: application/json" \
  -d '{"filename":"notes/daily","content":"# 日报\n\n今天完成了..."}'
```

---

### 2. 搜索文档 — `POST /open/search`

在全部文档中进行模糊搜索，支持多种匹配模式。

**请求 Body（JSON）：**

| 字段 | 类型 | 必填 | 默认 | 说明 |
|---|---|:---:|:---:|---|
| `q` | string | ✅ | — | 搜索词 |
| `mode` | string | — | `fuzzy` | 匹配模式（见下表） |
| `fields` | string | — | `both` | 搜索范围：`title` / `content` / `both` |
| `limit` | number | — | `20` | 最多返回结果数 |

**匹配模式 `mode`：**

| 值 | 说明 | 示例 |
|---|---|---|
| `fuzzy` | 模糊匹配，字符依次出现即命中（类 fzf） | `gtmd` 匹配 `getting-started.md` |
| `words` | 多词匹配，所有空格分隔的词均出现才命中 | `会议 记录` |
| `exact` | 精确子串匹配（忽略大小写） | `API` 匹配 `api` |
| `regex` | 正则表达式匹配 | `^\# .*接口` |

**示例：**

```bash
# 模糊搜索
curl -X POST http://localhost:7439/open/search \
  -H "Content-Type: application/json" \
  -d '{"q":"接口文档","mode":"words"}'

# 正则搜索
curl -X POST http://localhost:7439/open/search \
  -H "Content-Type: application/json" \
  -d '{"q":"^# .*指南","mode":"regex"}'
```

**响应示例：**

```json
{
  "success": true,
  "query": "接口文档",
  "mode": "words",
  "total": 2,
  "results": [
    {
      "path": "接口文档.md",
      "title": "对外接口文档",
      "score": 12,
      "excerpts": ["...本服务通过 `/open` 前缀暴露**对外开放接口**..."]
    }
  ]
}
```

### 3. 使用说明 — `GET /open/study`

返回本文档（如何使用文档）的 Markdown 源文本与 HTML 渲染结果，方便其他系统直接展示使用帮助。

```bash
curl http://localhost:7439/open/study
```

**响应：**

```json
{
  "success": true,
  "path": "如何使用文档.md",
  "content": "# 如何使用 DocSpace\n...",
  "html": "<h1>如何使用 DocSpace</h1>..."
}
```

## 三、快捷键

| 快捷键 | 功能 |
|---|---|
| `Ctrl + S` | 保存当前文档 |
| `Tab` | 编辑器中插入两个空格缩进 |
| `Escape` | 关闭弹窗 / 菜单 |
